# Task 2002: SNS アカウント接続管理

## 目的

SocialAccount の CRUD と OAuth 接続フロー（接続開始 URL 生成、コールバック処理、トークン暗号化保存）を実装する。最初は X (Twitter) の OAuth 2.0 PKCE フローを対象とする。

## 実装指示

1. **アカウント管理ユースケース** `core/usecases/account.ts` を実装する
   - `listAccounts(workspaceId)`: ワークスペースのアカウント一覧を返す（トークンは返さない）
   - `getAccount(accountId)`: アカウント詳細を返す
   - `initiateConnection(workspaceId, platform)`: Provider の `connectAccount` を呼び、OAuth URL を返す
   - `handleOAuthCallback(code, state)`: コールバックを処理し、トークンを暗号化して保存する
   - `disconnectAccount(accountId)`: アカウントを論理削除（status を 'revoked' に変更）する
   - `refreshAccountToken(accountId)`: Provider の `refreshToken` を呼び、新トークンで更新する
   - `checkTokenExpiry(workspaceId)`: 期限切れ間近のアカウントを検出して返す

2. **アカウント API エンドポイント** `routes/accounts.ts` の骨格を実装に置き換える
   - `GET /api/accounts` - listAccounts（RBAC: account:read）
   - `POST /api/accounts` - initiateConnection（RBAC: account:connect）。リクエストボディ: `{ platform }`。レスポンス: `{ data: { authUrl } }`
   - `GET /api/accounts/:id` - getAccount（RBAC: account:read）
   - `DELETE /api/accounts/:id` - disconnectAccount（RBAC: account:connect）
   - `POST /api/accounts/:id/refresh` - refreshAccountToken（RBAC: account:connect）
   - `GET /api/accounts/callback` - handleOAuthCallback（認証不要、state パラメータで検証）

3. **トークンの暗号化保存**
   - OAuth トークン（access_token, refresh_token）は `core/domain/crypto.ts` の encrypt で暗号化してから DB に保存する
   - `credentials_encrypted` カラムに JSON 文字列を暗号化して格納する
   - レスポンスにはトークンを含めない（status, expires_at のみ）

4. **トークン有効期限管理**
   - `token_expires_at` を保存し、API レスポンスに含める
   - 期限切れ間近（7日以内）のアカウントに `warning` フラグを付ける
