# Task 2006: 投稿一覧 API とフィルタリング

## 目的

全 SNS 横断の投稿一覧取得、プラットフォーム別フィルタ、ステータスフィルタ、日付範囲フィルタ、ページネーションを完成させる。Phase 2 の総合動作確認も含む。

## scope

- `apps/api/src/routes/posts.ts` - 一覧エンドポイントのフィルタ拡充
- `packages/db/src/repositories/post-repository.ts` - クエリ条件の追加
- `packages/core/src/usecases/post.ts` - listPosts の拡充

## 実装指示

1. **一覧クエリの拡充** PostRepository の `findByWorkspace` を拡充する
   - フィルタ条件: platform（複数選択可）、status（複数選択可）、from/to（日付範囲）、検索（contentText の部分一致）
   - ソート: created_at DESC（デフォルト）、published_at、scheduled_at で切り替え可能
   - ページネーション: page + limit（デフォルト 20、最大 100）
   - total count を返す

2. **API レスポンスの整形**
   - 投稿オブジェクトに関連情報を含める
     - `socialAccount`: { id, platform, displayName } を JOIN して返す
     - `schedule`: 予約がある場合は { scheduledAt, status } を含める
   - `meta`: { page, limit, total, totalPages }

3. **Phase 2 総合動作確認用テスト**
   - Vitest で以下のシナリオをテストする（統合テスト）
     - アカウント作成 -> 投稿作成（下書き） -> バリデーション -> 即時公開
     - アカウント作成 -> 投稿作成 -> 予約 -> （ワーカーによる実行は別途）
     - 文字数超過投稿の作成がバリデーションエラーになる
     - 投稿一覧のフィルタが正しく動作する
