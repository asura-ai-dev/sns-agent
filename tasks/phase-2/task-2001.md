# Task 2001: 認証・セッション管理と RBAC ミドルウェア

## 目的

API キー認証（CLI / SDK）とセッション認証（Web UI）の両方に対応する認証ミドルウェアと、RBAC に基づくアクセス制御ミドルウェアを実装する。

- packages/core/src/policies/rbac.ts - RBAC ポリシー定義

## scope

- `apps/api/src/middleware/auth.ts` - 認証ミドルウェア
- `apps/api/src/middleware/rbac.ts` - RBAC ミドルウェア
- `packages/core/src/usecases/auth.ts` - 認証ユースケース（API キーの検証、ユーザー解決）
- `packages/db/src/repositories/user-repository.ts` - User Repository 実装
- `packages/db/src/repositories/agent-identity-repository.ts` - AgentIdentity Repository 実装

## 実装指示

1. **認証ミドルウェア** `auth.ts` を実装する
   - `Authorization: Bearer <api_key>` ヘッダを検出した場合: API キーのハッシュを DB で検索し、AgentIdentity または User を解決する
   - セッション Cookie がある場合: セッションからユーザー情報を取得する
   - いずれもない場合: 401 UNAUTHORIZED を返す
   - 解決された actor 情報（id, type, role, workspace_id）を Hono context に `c.set("actor", actor)` でセットする

2. **RBAC ミドルウェア** `rbac.ts` を実装する
   - `requirePermission(permission: Permission)` ファクトリ関数として定義する
   - actor の role に基づいて core の `checkPermission()` を呼び出す
   - 権限なしの場合 403 FORBIDDEN を返す

3. **認証ユースケース** `core/usecases/auth.ts` を実装する
   - `resolveActorByApiKey(apiKeyHash)`: AgentIdentity を返す
   - `resolveActorByUserId(userId)`: User を返す
   - 両方とも workspace_id を含む actor 情報を返す

4. **User Repository** と **AgentIdentity Repository** を `packages/db` に実装する
   - core の対応するインターフェースを実装する
   - User: findById, findByEmail, findByWorkspace, create, update
   - AgentIdentity: findById, findByApiKeyHash, findByWorkspace, create, update, delete

5. **ワークスペース初期セットアップ**
   - 開発用に seed スクリプト（`packages/db/src/seed.ts`）を作成する
   - デフォルトワークスペース、owner ユーザー、agent identity を作成する
