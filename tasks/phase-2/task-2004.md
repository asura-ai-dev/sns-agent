# Task 2004: 投稿管理 - 作成・下書き・即時投稿・削除

## 目的

投稿の作成（下書き保存）、編集、即時公開、削除のユースケースと API エンドポイントを実装する。Provider のバリデーション連携と idempotency key による重複防止を含む。

## 実装指示

1. **投稿ユースケース** `core/usecases/post.ts` を実装する
   - `createPost(input)`: 投稿を作成する。`publishNow: true` なら即時公開、`false` なら下書き保存
     - 作成時に Provider の `validatePost` を呼び、バリデーション結果を `validation_result` に保存する
     - バリデーション失敗時は ValidationError を投げる（投稿は保存しない）
     - `idempotency_key` が指定されていれば重複チェックを行い、既存投稿を返す
   - `updatePost(postId, input)`: 下書きの編集。status が 'draft' でなければエラー
   - `publishPost(postId)`: 下書きを即時公開する
     - Provider の `publishPost` を呼び、成功時に `platform_post_id` と `published_at` を記録
     - 失敗時に status を 'failed' に更新し、エラー情報を保存
   - `deletePost(postId)`: 論理削除（status を 'deleted' に変更）。公開済みの場合は Provider の `deletePost` も呼ぶ
   - `listPosts(workspaceId, filters)`: フィルタ付き一覧取得（platform, status, 日付範囲）
   - `getPost(postId)`: 投稿詳細取得

2. **Idempotency ミドルウェア** `middleware/idempotency.ts` を実装する
   - `X-Idempotency-Key` ヘッダを検出する
   - POST/PATCH リクエストで idempotency key が既に処理済みなら、前回のレスポンスを返す
   - v1 では DB の `idempotency_key` カラムでの重複検出を最低限実装する

3. **投稿 API エンドポイント** `routes/posts.ts` を実装する
   - `GET /api/posts` - listPosts（RBAC: post:read）。クエリパラメータ: platform, status, from, to, page, limit
   - `POST /api/posts` - createPost（RBAC: post:create）。ボディ: `{ socialAccountId, contentText, contentMedia?, publishNow?, idempotencyKey? }`
   - `GET /api/posts/:id` - getPost（RBAC: post:read）
   - `PATCH /api/posts/:id` - updatePost（RBAC: post:create）
   - `DELETE /api/posts/:id` - deletePost（RBAC: post:publish）
   - `POST /api/posts/:id/publish` - publishPost（RBAC: post:publish）

4. **レスポンス形式**
   - 一覧: `{ data: Post[], meta: { page, limit, total } }`
   - 詳細: `{ data: Post }`
   - Post オブジェクトには `validationResult` を含める（UI でエラー表示に使用）
