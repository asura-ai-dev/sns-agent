# Task 2003: Provider 抽象と X (Twitter) プロバイダ実装

## 目的

SocialProvider インターフェースの実体として provider-x パッケージを実装する。X の OAuth 2.0 PKCE 認証、投稿のバリデーション・公開・削除、プロバイダ登録の仕組みを構築する。

## 実装指示

1. **プロバイダ登録機構** `core/interfaces/provider-registry.ts` を作成する
   - `ProviderRegistry` クラス: `register(provider)`, `get(platform)`, `getAll()` メソッド
   - apps/api の初期化時に各 provider を登録する

2. **X プロバイダ** `packages/provider-x/src/` を実装する
   - `index.ts`: SocialProvider インターフェースを実装するクラスを export
   - `auth.ts`: OAuth 2.0 PKCE フローの実装
     - `getAuthUrl(redirectUri, state)`: 認可 URL を生成
     - `exchangeCode(code, codeVerifier)`: access_token + refresh_token を取得
     - `refreshToken(refreshToken)`: トークンをリフレッシュ
   - `post.ts`: 投稿操作
     - `validatePost`: テキスト文字数チェック（280文字 / 25,000文字）、画像枚数チェック等
     - `publishPost`: X API v2 の POST /2/tweets を呼び出し
     - `deletePost`: DELETE /2/tweets/:id を呼び出し
   - `capabilities.ts`: X の capability 定義

3. **X API クライアント**
   - X API v2 のエンドポイントを呼ぶ薄い HTTP クライアントを内部に持つ
   - レート制限ヘッダ（x-rate-limit-remaining 等）を解析して返す
   - API エラーを ProviderError に変換する

4. **capability 定義**

   ```
   textPost: true
   imagePost: true
   videoPost: true
   threadPost: false  // v1 ではスレッド非対応
   directMessage: true
   commentReply: true
   broadcast: false
   nativeSchedule: false  // X API v2 にネイティブ予約なし
   usageApi: false
   ```

5. `apps/api/src/providers.ts` で X プロバイダを ProviderRegistry に登録する
