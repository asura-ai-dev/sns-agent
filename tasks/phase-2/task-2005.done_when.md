# Task 2005 done_when

- `POST /api/schedules` に `{ postId, scheduledAt }` を送ると予約ジョブが作成され、投稿の status が "scheduled" になる
- 過去の日時を指定した場合に 400 バリデーションエラーが返る
- `PATCH /api/schedules/:id` で予約日時を変更できる
- `DELETE /api/schedules/:id` で予約がキャンセルされ、投稿の status が "draft" に戻る
- `GET /api/schedules` で予約一覧を取得でき、status でフィルタできる
- Polling ワーカーが `scheduled_at <= now()` のジョブを検出して実行する
- ジョブ実行成功時に status が "succeeded" になり、投稿が公開される
- ジョブ実行失敗時に attempt_count がインクリメントされ、max_attempts 未満なら "retrying" になる
- 3 回失敗後に status が "failed" になり、last_error にエラー内容が記録される
- 再試行の間隔が exponential backoff（30s, 120s, 480s）に従っている
- `ENABLE_SCHEDULER=false` の場合にワーカーが起動しない
