## 実装指示

1. **API 統合テスト** `apps/api/src/__tests__/integration/`
   - vitest + supertest で API サーバーに対する統合テストを作成する
   - テストシナリオ:
     a. **アカウント接続フロー**: POST /api/accounts -> GET /api/accounts -> アカウント一覧に反映
     b. **投稿作成 -> 公開フロー**: POST /api/posts (draft) -> POST /api/posts/:id/publish -> GET /api/posts/:id (published)
     c. **予約投稿フロー**: POST /api/posts (draft) -> POST /api/schedules -> GET /api/schedules/:id (pending)
     d. **バリデーションエラー**: 280 文字超過の X 投稿 -> 400 エラー
     e. **RBAC**: viewer が POST /api/posts -> 403、editor が POST /api/posts -> 201
     f. **Idempotency**: 同じ X-Idempotency-Key で 2 回 POST -> 2 回目は既存を返す
     g. **予算チェック**: 予算超過状態で投稿公開 -> block ポリシーなら 403
     h. **承認フロー**: agent ロールが公開 -> 承認リクエスト作成 -> admin が承認 -> 公開実行
     i. **監査ログ**: 操作後に GET /api/audit で記録が存在する
     j. **使用量記録**: 投稿公開後に GET /api/usage で記録が存在する

2. **CLI 統合テスト** `packages/cli/src/__tests__/`
   - テスト API サーバーに対して CLI コマンドを実行するテスト
   - テストシナリオ:
     a. `sns accounts list --json` が JSON を返し終了コード 0
     b. `sns post create --platform x --account test --text "Hello" --json` が投稿を作成
     c. `sns schedule create --post <id> --at "..." ` が予約を作成
     d. `sns usage report --range month --json` が使用量を返す
     e. 不正なコマンドで終了コード 1

3. **Web UI E2E テスト（最小限）** `apps/web/src/__tests__/e2e/`
   - Playwright で最低限のスモークテストを作成する
   - テストシナリオ:
     a. ダッシュボードにアクセスしてサマリカードが表示される
     b. 投稿一覧にアクセスしてテーブルが表示される
     c. 投稿作成フォームでテキストを入力して下書き保存できる
     d. カレンダーにアクセスしてカレンダーが表示される
     e. 設定画面でアカウント一覧が表示される

4. **テスト実行環境**
   - テスト用 SQLite DB を使用する（各テストスイートで初期化）
   - テスト用 seed データ: ワークスペース、owner ユーザー、viewer ユーザー、agent identity、X アカウント（モック）
   - 外部 API（X, LINE, Instagram, OpenAI, Anthropic）はモック化する
