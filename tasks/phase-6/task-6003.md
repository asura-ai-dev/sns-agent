# Task 6003: 受信・会話管理（Inbox）

## 目的

X リプライ/DM、LINE チャット、Instagram DM/コメントを統一的に参照する受信トレイ機能を実装する。Webhook 経由でメッセージを蓄積し、Web UI と CLI から閲覧できるようにする。

## 実装指示

1. **Inbox ユースケース** `core/usecases/inbox.ts`
   - `listThreads(workspaceId, filters)`: スレッド一覧取得
     - filters: platform, status(open/closed/archived), limit, offset
     - 最新メッセージ日時でソート
   - `getThread(threadId)`: スレッド詳細 + メッセージ一覧
   - `processInboundMessage(event)`: Webhook から受信したメッセージを処理する
     - スレッドが存在しなければ作成
     - メッセージを messages テーブルに保存
     - スレッドの last_message_at を更新
   - `sendReply(threadId, content, actorId)`: 返信を送信する
     - 承認フロー（Phase 6 の承認ポリシー）を経由する
     - Provider の `sendReply` を呼ぶ
     - messages テーブルに outbound として記録

2. **Repository 実装**
   - `ConversationRepository`: findByWorkspace, findById, create, update
   - `MessageRepository`: findByThread, create

3. **Webhook ハンドラ拡充** `routes/webhooks.ts`
   - 各プラットフォームの Webhook エンドポイントで受信イベントを解析し、`processInboundMessage` に渡す
   - X: mention, reply, DM イベント
   - LINE: message event, follow event
   - Instagram: messaging webhook, comment webhook

4. **Inbox API** `routes/inbox.ts`
   - `GET /api/inbox` - スレッド一覧（RBAC: inbox:read）
     - クエリパラメータ: platform, status, limit, offset
   - `GET /api/inbox/:threadId` - メッセージ一覧（RBAC: inbox:read）
     - limit, offset でページネーション
   - `POST /api/inbox/:threadId/reply` - 返信送信（RBAC: inbox:reply -> editor 以上）

5. **受信トレイ画面** `inbox/page.tsx`
   - 2 カラムレイアウト
     - 左: スレッド一覧（SNS アイコン、相手名、最新メッセージ冒頭、日時、未読バッジ）
     - 右: 選択中スレッドのメッセージ一覧 + 返信入力
   - プラットフォームフィルタ（タブ: 全て / X / LINE / Instagram）
   - メッセージバブル: inbound（左寄せ）/ outbound（右寄せ）
   - 返信入力: テキストエリア + 送信ボタン
   - モバイルでは左パネル/右パネルのスライド切り替え
