# Task 6002: 承認フロー

## 目的

書き込み操作に対する承認ワークフローを実装する。承認リクエストの作成、承認/却下、承認待ちリストの表示、予算超過時の承認連携を構築する。

## 実装指示

1. **承認ポリシー** `core/policies/approval.ts`
   - `requiresApproval(action, actorRole, context)`: 操作が承認を必要とするか判定する
   - デフォルトで承認が必要な操作:
     - agent ロールによる投稿公開
     - 予算超過（action_on_exceed = 'require-approval'）時の操作続行
     - broadcast（LINE 全友だち配信）
   - admin/owner は承認不要（direct-execute モード）
   - 承認ルールはワークスペース設定として保存可能にする

2. **承認ユースケース** `core/usecases/approval.ts`
   - `createApprovalRequest(input)`: 承認リクエスト作成
     - input: workspaceId, resourceType, resourceId, requestedBy, reason
     - status: 'pending'
   - `approveRequest(requestId, reviewerId)`: 承認
     - status を 'approved' に更新
     - 承認後に元の操作を実行する（投稿公開等）
     - 監査ログに記録
   - `rejectRequest(requestId, reviewerId, reason)`: 却下
     - status を 'rejected' に更新
     - 監査ログに記録
   - `listPendingApprovals(workspaceId)`: 承認待ちリスト
   - `expireStaleRequests()`: 一定期間（24h）経過した pending を expired に更新

3. **承認 API エンドポイント** `routes/approvals.ts`
   - `GET /api/approvals` - 承認リクエスト一覧（RBAC: approval:read -> admin 以上）
     - クエリパラメータ: status(pending/approved/rejected/expired), page, limit
   - `POST /api/approvals/:id/approve` - 承認（RBAC: approval:manage -> admin 以上）
   - `POST /api/approvals/:id/reject` - 却下（RBAC: approval:manage）。ボディ: `{ reason? }`

4. **投稿フローとの統合**
   - `publishPost` ユースケースで `requiresApproval` を呼び、true の場合:
     - 承認リクエストを作成する
     - 投稿の status を 'scheduled'（承認待ち）に変更する
     - レスポンスに `{ requiresApproval: true, approvalId }` を含める
   - 承認後に `publishPost` を再実行する

5. **Web UI の承認 UI**
   - Header の通知ベルに承認待ち件数のバッジを表示する
   - 通知ドロップダウンに承認待ちリストを表示する
   - 承認/却下ボタンを各リスト項目に配置する
   - 承認ダイアログで対象の詳細（投稿内容、操作種別、リクエスト者）を表示する
