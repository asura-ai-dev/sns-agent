# Task 5006: Agent Gateway invoker 実装（Phase 5 fix）

## 目的

Phase 5 evaluator の FAIL を修正する。Agent Gateway の `executeInvoker` が `placeholderInvoker` のままで core use case に委譲していないため、チャットから承認済みアクションを実行しても実投稿が行われない。action 名から core use case への実マッピングを実装する。

[5002, 5003]

- tasks/phase-5/task-5002.md
- packages/skills/src/builder/templates.ts - ビルトインアクションテンプレート
- packages/core/src/usecases/post.ts - createPost, publishPost, listPosts
- packages/core/src/usecases/schedule.ts - schedulePost

## scope

- `apps/api/src/routes/agent.ts` - placeholderInvoker を実マッピングに差し替え

## 実装指示

1. `apps/api/src/routes/agent.ts` に action 名から core use case へのディスパッチ関数を追加
2. packages/skills/src/builder/templates.ts のビルトインアクション（list_accounts, list_posts, create_post, schedule_post）を対象とする
3. skill manifest の各 action に対応する use case を呼び、args をマッピングする
   - list_accounts → listAccounts(accountDeps, { workspaceId })
   - list_posts → listPosts(postDeps, { workspaceId, filters })
   - create_post → createPost(postDeps, { workspaceId, authorId, socialAccountId, contentText, publishNow })
   - schedule_post → schedulePost(scheduleDeps, { workspaceId, postId or createPost + scheduleAt })
4. 未知の action 名は `{ status: "unsupported_action", actionName }` を返す
5. 戻り値の Record<string, unknown> に作成物の id / status を詰める
6. 既存の placeholderInvoker を削除

## 実装ヒント

- socialAccountId は skill action args に含まれるか、workspaceId + platform から解決する
- accountRepo / postRepo / scheduledJobRepo は buildDeps() から取得する
- providerRegistry も必要（createPost/publishPost の deps に渡す）
- 引数マッピングが不明なパラメータは action args の一部を直接使う
