# Task 5005: Web UI - LLM ルーティング設定画面と Skills 管理画面

## 目的

LLM ルーティング設定の UI と Skills パッケージ管理の UI を実装する。

## depends_on

[3004, 5001, 5003]

## refs

- agent-docs/design.md - セクション 10.2（Skills 管理画面、設定画面）
- agent-docs/spec.md - AC-18, AC-20

## scope

- `apps/web/src/app/(dashboard)/settings/llm/page.tsx` - LLM ルーティング設定ページ
- `apps/web/src/app/(dashboard)/skills/page.tsx` - Skills 管理ページ
- `apps/web/src/components/settings/llm/` - LLM 設定コンポーネント
- `apps/web/src/components/skills/` - Skills コンポーネント

## 実装指示

1. **LLM ルーティング設定** `settings/llm/page.tsx`
   - 設定画面のタブに「LLM ルーティング」を追加する
   - ルート一覧テーブル
     - 各行: Platform（アイコン付き）、Action、Provider、Model、Temperature、Max Tokens、Fallback、Priority
     - デフォルトルート（platform/action が null）は「デフォルト」と表示
   - ルート作成/編集フォーム（モーダル）
     - Platform: ドロップダウン（全体 / X / LINE / Instagram）
     - Action: ドロップダウン（全体 / create_post / schedule_post / reply 等）
     - Provider: ドロップダウン（openai / anthropic）
     - Model: テキスト入力（gpt-4o, claude-sonnet-4-20250514 等）
     - Temperature: スライダー（0.0 - 2.0）
     - Max Tokens: 数値入力
     - Fallback Provider + Model: 任意
   - ルート削除（確認ダイアログ付き）

2. **Skills 管理** `skills/page.tsx`
   - パッケージ一覧
     - 各行: パッケージ名、Platform アイコン、LLM Provider、Version、有効/無効トグル、アクション数
   - パッケージ生成ボタン
     - Platform 選択 + LLM Provider 選択のダイアログ
     - 生成中はローディング表示
     - 生成完了後に一覧に追加
   - パッケージ詳細（クリックで展開またはモーダル）
     - Manifest の内容を表示: 名前、バージョン、アクション一覧
     - 各アクション: 名前、説明、パラメータ（JSON Schema のフレンドリー表示）、必要権限
   - 有効/無効トグル: DaisyUI の toggle コンポーネント

3. **空状態**
   - ルートが 0 件: 「LLM ルーティングが未設定です。デフォルトルートを作成してください」
   - パッケージが 0 件: 「Skills パッケージがありません。パッケージを生成してください」
