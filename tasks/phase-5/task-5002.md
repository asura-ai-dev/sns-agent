# Task 5002: Agent Gateway と skill 実行基盤

## 目的

Web UI チャットと CLI から LLM を呼び出す Agent Gateway を構築する。skill manifest に基づく実行制御、権限検証、予算検証、dry-run プレビューを実装する。

## 実装指示

1. **Skill Manifest 型定義** `packages/skills/src/manifest/types.ts`
   - `SkillManifest`: name, version, platform, provider, description
   - `SkillAction`: name, description, parameters (JSON Schema), permissions[], requiredCapabilities[]
   - `SkillManifest.actions: SkillAction[]`
   - マニフェストのバリデーション関数を含める

2. **Skill Executor** `packages/skills/src/runtime/executor.ts`
   - `validateSkillAction(manifest, actionName, args)`: 引数を JSON Schema でバリデーション
   - `checkSkillPermissions(manifest, actionName, actorRole)`: 権限チェック
   - `executeSkillAction(context)`: skill アクションを実行する
     - context: manifest, actionName, args, actor, workspace
     - 実行前に権限チェック、予算チェックを行う
     - 実際の実行は core のユースケース（createPost, schedulePost 等）に委譲する
   - `dryRunSkillAction(context)`: 実行せずにプレビュー（何が実行されるかの説明）を返す

3. **Agent Gateway ユースケース** `core/usecases/agent-gateway.ts`
   - `handleChatMessage(workspaceId, actorId, message)`:
     - LLM Route Resolver でモデルを選択
     - ワークスペースの有効な skills から system prompt を構築（利用可能なアクション一覧を含める）
     - LLM を呼び出し、レスポンスを解析
     - LLM が skill 実行を要求した場合: dry-run プレビューを生成して返す
     - LLM がテキスト応答のみの場合: そのまま返す
   - `executeAgentAction(workspaceId, actorId, actionRequest)`:
     - 承認済みのアクション実行リクエストを受け取る
     - Skill Executor に委譲して実行する
     - 監査ログを記録する

4. **Agent Gateway API** `routes/agent.ts`
   - `POST /api/agent/chat` - チャットメッセージ送信（RBAC: chat:use）
     - ボディ: `{ message, conversationId? }`
     - レスポンス: SSE ストリーム（テキスト応答の場合）またはアクションプレビュー JSON
   - `POST /api/agent/execute` - 承認済みアクション実行（RBAC: chat:use + アクション個別の権限）
     - ボディ: `{ actionName, args, conversationId }`
   - `GET /api/agent/history` - チャット履歴（RBAC: chat:use）

5. **実行モード**（design.md セクション 8.5 準拠）
   - v1 デフォルト: `approval-required`（全 skill アクションは dry-run -> 承認 -> 実行）
   - `read-only`: 読み取り系アクションのみ許可
   - `draft`: 書き込みは下書きのみ
   - `direct-execute`: 承認不要で即時実行（admin/owner 向け設定）
