## 実装指示

1. **パッケージビルダー** `packages/skills/src/builder/`
   - `generateSkillPackage(platform, llmProvider)`: platform と LLM provider の組み合わせから skill パッケージを生成する
     - ProviderRegistry から capability を取得
     - capability に応じて利用可能なアクション（createPost, schedulePost, listPosts 等）を決定
     - 各アクションの引数スキーマを JSON Schema で定義
     - manifest JSON を生成
   - ビルトインテンプレート: X, LINE, Instagram のそれぞれに対して基本的なアクション定義を内蔵する
   - 生成された manifest は `skill_packages` テーブルに保存する

2. **Manifest パーサー・バリデーター** `packages/skills/src/manifest/`
   - `parseManifest(json)`: JSON から SkillManifest を解析する
   - `validateManifest(manifest)`: 必須フィールド、アクション定義、パーミッション定義を検証する
   - バージョン互換性チェック

3. **Skills ユースケース** `core/usecases/skills.ts`
   - `listSkillPackages(workspaceId)`: パッケージ一覧
   - `generateSkillPackage(workspaceId, platform, llmProvider)`: パッケージ生成
   - `enableSkillPackage(packageId)`: パッケージ有効化
   - `disableSkillPackage(packageId)`: パッケージ無効化
   - `getSkillManifest(packageId)`: manifest 取得
   - `getActiveSkills(workspaceId)`: 有効な全 skill の manifest を返す（Agent Gateway が参照する）

4. **Skills API エンドポイント** `routes/skills.ts`
   - `GET /api/skills` - パッケージ一覧（RBAC: skills:read -> admin 以上）
   - `POST /api/skills/generate` - パッケージ生成（RBAC: skills:manage）。ボディ: `{ platform, llmProvider }`
   - `PATCH /api/skills/:id` - 有効化/無効化（RBAC: skills:manage）。ボディ: `{ enabled }`
   - `GET /api/skills/:id/manifest` - manifest 取得（RBAC: skills:read）

5. **ビルトインアクション定義例**（X の場合）
   - `create_post`: テキスト投稿作成。params: { text, publishNow }
   - `schedule_post`: 投稿予約。params: { text, scheduledAt }
   - `list_posts`: 投稿一覧取得。params: { status?, limit? }
   - `list_accounts`: アカウント一覧取得。params: {}
