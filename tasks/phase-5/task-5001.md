## 実装指示

1. **LLM アダプタインターフェース** `types.ts`
   - `LlmAdapter` インターフェース: `chat(messages, options)`, `stream(messages, options)`
   - `ChatMessage`: `{ role: 'system'|'user'|'assistant', content: string }`
   - `ChatOptions`: `{ model, temperature, maxTokens, timeout }`
   - `ChatResponse`: `{ content: string, usage: { promptTokens, completionTokens, totalTokens }, model }`

2. **OpenAI アダプタ** `adapters/openai.ts`
   - OpenAI API の chat completions を呼ぶ
   - `openai` npm パッケージを使用する
   - ストリーミング対応（SSE 用）
   - エラーを LlmError に変換する

3. **Anthropic アダプタ** `adapters/anthropic.ts`
   - Anthropic API の messages を呼ぶ
   - `@anthropic-ai/sdk` npm パッケージを使用する
   - ストリーミング対応
   - エラーを LlmError に変換する

4. **Route Resolver** `router.ts`
   - `resolveLlmRoute(workspaceId, platform?, action?)`: llm_routes テーブルから最適なルートを解決する
     - 優先度: platform + action > platform のみ > デフォルト（platform=null, action=null）
     - 解決されたルートからアダプタとオプションを返す
   - `executeLlmCall(route, messages)`: ルートに基づいて LLM を呼び出す
     - メインモデルで失敗した場合、fallback_provider + fallback_model で再試行する
     - 使用量を recordUsage で記録する（トークン数から推定コストを計算）

5. **LlmRouteRepository 実装**
   - `findByWorkspace(workspaceId)`: 一覧
   - `resolve(workspaceId, platform?, action?)`: ルート解決（priority 順）
   - `create`, `update`, `delete`

6. **LLM ルーティング API** `routes/llm.ts`
   - `GET /api/llm/routes` - ルート一覧（RBAC: llm:read -> admin 以上）
   - `POST /api/llm/routes` - ルート作成（RBAC: llm:manage）
   - `PATCH /api/llm/routes/:id` - ルート更新（RBAC: llm:manage）
   - `DELETE /api/llm/routes/:id` - ルート削除（RBAC: llm:manage）
