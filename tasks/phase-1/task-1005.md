# Task 1005: packages/sdk - TypeScript SDK 基盤

## 目的

CLI と Web UI が共通で利用する API クライアント SDK を実装する。REST API のリクエスト/レスポンス型、HTTP クライアントラッパー、エラーハンドリングを提供する。

[1002, 1003]

## refs

- agent-docs/design.md - セクション 4（API エンドポイント設計）、セクション 4.3（共通レスポンス形式）
- docs/architecture.md - セクション 5.4（packages/sdk）

## scope

- `packages/sdk/src/client.ts` - HTTP クライアント（fetch ベース）
- `packages/sdk/src/types.ts` - API リクエスト/レスポンス型
- `packages/sdk/src/errors.ts` - SDK エラー型
- `packages/sdk/src/index.ts` - エクスポート

## 実装指示

1. **SnsAgentClient クラス**を `client.ts` に実装する
   - コンストラクタ: `baseUrl`, `apiKey` を受け取る
   - 内部で `fetch` を使い、共通ヘッダ（Authorization, Content-Type, X-Idempotency-Key）を付与する
   - メソッド: `get<T>(path, params?)`, `post<T>(path, body?)`, `patch<T>(path, body?)`, `delete<T>(path)` の汎用メソッド
   - レスポンスが 4xx/5xx の場合は `SdkError` をスローする

2. **リソース別メソッド**を SnsAgentClient に定義する（型だけ定義し、実装は薄いラッパー）
   - `accounts.list()`, `accounts.get(id)`, `accounts.connect(input)`, `accounts.disconnect(id)`
   - `posts.list(params)`, `posts.get(id)`, `posts.create(input)`, `posts.update(id, input)`, `posts.delete(id)`, `posts.publish(id)`
   - `schedules.list(params)`, `schedules.create(input)`, `schedules.update(id, input)`, `schedules.cancel(id)`
   - `usage.report(params)`, `usage.summary()`

3. **型定義** `types.ts` に API のリクエスト/レスポンス型を定義する
   - design.md セクション 4.3 の `{ data, meta }` / `{ error }` 形式に準拠する
   - core のドメインエンティティ型を再利用可能な箇所は re-export する

4. **エラー型** `errors.ts` に `SdkError` を定義する
   - `statusCode`, `code`, `message`, `details` を持つ
