# Task 1001: モノレポ初期化とプロジェクトセットアップ

## 目的

pnpm workspaces + turborepo によるモノレポ骨格を構築し、全パッケージのディレクトリ構造と共通設定を整備する。

## depends_on

なし

## refs

- agent-docs/design.md - セクション 1.9（モノレポツール）、セクション 2（ディレクトリ構成）

## scope

- ルートの `package.json`, `pnpm-workspace.yaml`, `turbo.json`, `tsconfig.json`
- 全パッケージ・アプリの `package.json` と `tsconfig.json`（中身は空の骨格）
- `.npmrc`, `.gitignore` 更新
- ESLint / Prettier 共通設定

## 実装指示

1. ルートに `pnpm-workspace.yaml` を作成し、`apps/*` と `packages/*` を workspace として定義する
2. `turbo.json` を作成し、`build`, `dev`, `lint`, `test` のパイプラインを定義する
3. 以下のパッケージ/アプリの骨格ディレクトリと `package.json` を作成する
   - `apps/web` - Next.js アプリ（`next`, `react`, `react-dom` を依存に含む）
   - `apps/api` - Hono API サーバー（`hono`, `@hono/node-server` を依存に含む）
   - `packages/core` - ドメインロジック（依存なし）
   - `packages/db` - DB スキーマ（`drizzle-orm` を依存に含む）
   - `packages/sdk` - TypeScript SDK
   - `packages/cli` - CLI
   - `packages/ui` - 共通 UI コンポーネント
   - `packages/llm` - LLM アダプタ
   - `packages/skills` - skills パッケージ機構
   - `packages/provider-x` - X プロバイダ
   - `packages/provider-line` - LINE プロバイダ
   - `packages/provider-instagram` - Instagram プロバイダ
   - `packages/config` - 共通設定
4. ルート `tsconfig.json` を base config として作成し、各パッケージは `extends` で参照する
5. 各パッケージの `package.json` に `name` を `@sns-agent/<パッケージ名>` で設定する
6. `pnpm install` が正常完了する状態にする
7. ESLint は `@typescript-eslint` を使い、Prettier と連携させる
8. ルート `package.json` の `scripts` に `build`, `dev`, `lint`, `test`, `format` を定義する
