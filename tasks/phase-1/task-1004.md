# Task 1004: packages/db - Drizzle スキーマとリポジトリ実装

## 目的

Drizzle ORM を使った DB スキーマ定義、マイグレーション基盤、SQLite/PostgreSQL 切り替え可能な接続ファクトリ、core の Repository インターフェースの実装を行う。

## depends_on

[1003]

## refs

- agent-docs/design.md - セクション 1.1（DB / ORM）、セクション 3（DB スキーマ設計）、セクション 3.2（インデックス戦略）
- packages/core/src/interfaces/ - 全 Repository インターフェース

## scope

- `packages/db/src/schema/` - 全テーブルの Drizzle スキーマ定義
- `packages/db/src/client.ts` - DB 接続ファクトリ（SQLite / PostgreSQL 切り替え）
- `packages/db/src/repositories/` - Repository 実装
- `packages/db/drizzle.config.ts` - Drizzle Kit 設定
- `packages/db/src/index.ts` - エクスポート

## 実装指示

1. **DB 接続ファクトリ** `client.ts` を実装する
   - `DATABASE_URL` が `file:` で始まる場合は better-sqlite3 ドライバ、`postgres://` の場合は postgres ドライバを使う
   - `getDb()` 関数として export する
   - drizzle-orm の `drizzle()` を使って接続を初期化する

2. **スキーマ定義** `schema/` に各テーブルを定義する
   - design.md セクション 3.1 の全テーブルを Drizzle のテーブル定義に変換する
   - テーブルごとにファイルを分ける（`workspaces.ts`, `users.ts`, `social-accounts.ts`, `posts.ts`, `scheduled-jobs.ts`, `conversation-threads.ts`, `messages.ts`, `usage-records.ts`, `budget-policies.ts`, `llm-routes.ts`, `skill-packages.ts`, `approval-requests.ts`, `audit-logs.ts`）
   - `schema/index.ts` で全テーブルを re-export する
   - enum 値は `@sns-agent/config` の定数を参照する
   - design.md セクション 3.2 のインデックスを定義する

3. **Repository 実装** `repositories/` に core の Repository インターフェースを実装する
   - Phase 1 では最低限 `AccountRepository`, `PostRepository`, `ScheduledJobRepository` を実装する
   - 残りの Repository は後続フェーズで追加するが、ファイルの骨格（クラス + 未実装メソッド）は作成する

4. **Drizzle Kit 設定** `drizzle.config.ts` を作成する
   - スキーマパス、マイグレーション出力先を設定する

5. **マイグレーション** `pnpm drizzle-kit generate` でマイグレーションファイルを生成し、`pnpm drizzle-kit push` でスキーマを適用できる状態にする

6. `package.json` の scripts に `db:generate`, `db:push`, `db:studio` を定義する
