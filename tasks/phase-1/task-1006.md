# Task 1006: apps/api - Hono API サーバー骨格

## 目的

Hono ベースの API サーバーの骨格を構築する。ルーティング構造、共通ミドルウェア（リクエストID、エラーハンドリング、CORS）、ヘルスチェックエンドポイントを実装し、`pnpm dev` で起動できる状態にする。

[1002, 1003, 1004]

## refs

- agent-docs/design.md - セクション 1.2（API スタイル）、セクション 1.4（実行基盤）、セクション 4.4（ミドルウェアチェーン）
- docs/architecture.md - セクション 5.2（apps/api）

## scope

- `apps/api/src/index.ts` - サーバーエントリポイント
- `apps/api/src/app.ts` - Hono アプリ初期化
- `apps/api/src/routes/` - ルーティングファイル群（骨格のみ）
- `apps/api/src/middleware/` - 共通ミドルウェア

## 実装指示

1. **エントリポイント** `index.ts` で `@hono/node-server` の `serve()` を使い、ポート `API_PORT`（デフォルト 3001）で起動する

2. **Hono アプリ** `app.ts` で Hono インスタンスを生成し、ミドルウェアとルーティングを組み立てる

3. **共通ミドルウェア** `middleware/` に以下を実装する
   - `request-id.ts`: `X-Request-Id` ヘッダを付与（UUID v4）。既存ヘッダがあればそれを使う
   - `error-handler.ts`: 未処理エラーを捕捉し、design.md セクション 4.3 のエラーレスポンス形式で返す。DomainError はそのまま変換、それ以外は 500 + SYSTEM_ERROR
   - `cors.ts`: Hono の cors ミドルウェアを設定（開発環境は `*`、本番は `WEB_URL` のみ許可）

4. **ルーティング骨格** `routes/` に以下のファイルを作成する。各ファイルに Hono の Router を定義し、ダミーの 501 レスポンスを返す最低 1 エンドポイントを置く
   - `accounts.ts`, `posts.ts`, `schedules.ts`, `usage.ts`, `budget.ts`, `llm.ts`, `skills.ts`, `agent.ts`, `audit.ts`, `approvals.ts`, `webhooks.ts`

5. **ヘルスチェック** `GET /api/health` で `{ status: "ok" }` を返すエンドポイントを実装する

6. `apps/api/package.json` の scripts に `dev`（tsx watch）と `build` を定義する

7. DB 接続を app 初期化時に `@sns-agent/db` の `getDb()` で取得し、Hono の context に注入する方式を用意する（c.set / c.get パターン）
