# Task 3001: packages/cli - CLI 基盤と accounts コマンド

## 目的

CLI の基盤（コマンドパーサー、出力フォーマッター、設定読み込み）を構築し、最初のコマンドとして `sns accounts` を実装する。

## scope

- `packages/cli/src/index.ts` - エントリポイント
- `packages/cli/src/commands/accounts.ts` - accounts コマンド
- `packages/cli/src/formatters/` - JSON / human-readable フォーマッター
- `packages/cli/src/config.ts` - CLI 設定（API URL, API Key）読み込み
- `packages/cli/package.json` - bin フィールド設定

## 実装指示

1. **CLI 基盤**
   - コマンドパーサーとして `commander` を使用する
   - エントリポイント: `sns` コマンド（`bin` フィールドで `sns` にマッピング）
   - グローバルオプション: `--json`（JSON 出力）、`--api-url`（API URL）、`--api-key`（API キー）
   - 設定の読み込み優先度: コマンドライン引数 > 環境変数（`SNS_API_URL`, `SNS_API_KEY`）> `~/.sns-agent/config.json`
   - 終了コード: 成功 = 0、エラー = 1

2. **出力フォーマッター** `formatters/` を実装する
   - `json.ts`: `--json` 指定時に JSON を標準出力する
   - `table.ts`: デフォルトの人間可読テーブル出力
   - フォーマッターは共通インターフェースを持ち、コマンドから切り替え可能にする

3. **accounts コマンド** `commands/accounts.ts` を実装する
   - `sns accounts list` - 接続済みアカウント一覧を表示する
     - `--json` でJSON出力、それ以外はテーブル出力
     - `--platform <p>` でフィルタ可能
   - `sns accounts show <id>` - アカウント詳細を表示する
   - `sns accounts connect <platform>` - OAuth URL を表示し、ブラウザで開くよう案内する
   - `sns accounts disconnect <id>` - アカウントを切断する
   - 全コマンドは SDK の SnsAgentClient を内部利用する

4. **エラーハンドリング**
   - SDK からの SdkError を catch し、人間可読なエラーメッセージを stderr に出力する
   - `--json` の場合はエラーも JSON 形式で stdout に出力する
   - 全エラーで終了コード 1 を返す
