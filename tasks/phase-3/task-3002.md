## 実装指示

1. **post コマンド** `commands/post.ts`
   - `sns post list` - 投稿一覧。`--platform`, `--status`, `--limit`, `--json`
   - `sns post create` - 投稿作成
     - `--platform <p>` （必須）
     - `--account <name|id>` （必須）
     - `--text "..."` または `--file <path>`（ファイルから本文読み込み）
     - `--media <path>` （複数指定可）
     - `--publish` （即時公開。省略時は下書き）
     - `--json`
   - `sns post show <id>` - 投稿詳細。`--json`
   - `sns post delete <id>` - 投稿削除
   - `sns post publish <id>` - 下書きを即時公開

2. **schedule コマンド** `commands/schedule.ts`
   - `sns schedule list` - 予約一覧。`--status`, `--json`
   - `sns schedule create` - 予約作成。`--post <id>` `--at "2026-04-12T09:00:00+09:00"`
   - `sns schedule show <id>` - 予約詳細。`--json`
   - `sns schedule cancel <id>` - 予約キャンセル
   - `sns schedule update <id>` - 予約日時変更。`--at "..."`

3. **inbox コマンド** `commands/inbox.ts`
   - `sns inbox list` - スレッド一覧。`--platform`, `--limit`, `--json`
   - `sns inbox show <threadId>` - メッセージ一覧。`--limit`, `--json`
   - 注: Phase 2 時点では inbox API が未実装の場合、SDK 呼び出しの骨格のみ実装し、API 未実装エラーを適切にハンドリングする

4. **共通パターン**
   - 全コマンドで `--json` を統一的にサポートする
   - 入力バリデーションはコマンド側でも行う（必須パラメータチェック）
   - SDK の SnsAgentClient を使って API を呼ぶ
