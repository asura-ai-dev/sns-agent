# Task 3004: Web UI - Next.js 初期化とレイアウト・テーマ

## 目的

apps/web の Next.js アプリを初期化し、DaisyUI テーマ、共通レイアウト（AppShell: Sidebar + Header）、レスポンシブナビゲーションを構築する。

- docs/requirements.md - セクション 7.6（Web UI、ナビゲーション要件、デザイン要件）

## scope

- `apps/web/` - Next.js アプリ全体の初期化
- `apps/web/tailwind.config.ts` - DaisyUI テーマ定義
- `apps/web/src/app/layout.tsx` - ルートレイアウト
- `apps/web/src/components/layout/` - Sidebar, Header, AppShell

## 実装指示

1. **Next.js 初期化**
   - App Router を使用する
   - `next.config.ts` を設定する（standalone 出力、transpilePackages に workspace パッケージを追加）
   - `@sns-agent/ui`, `@sns-agent/sdk`, `@sns-agent/config` を依存に追加する

2. **Tailwind CSS + DaisyUI 設定**
   - `tailwind.config.ts` に DaisyUI プラグインを追加する
   - design.md セクション 10.3 のカスタムテーマ `sns-agent` を定義する
   - フォント: system-ui を基本とする
   - `@phosphor-icons/react` をインストールする

3. **共通レイアウト** `components/layout/` を実装する
   - `AppShell.tsx`: Sidebar + Header + Main の3カラム構成
   - `Sidebar.tsx`: ナビゲーションメニュー
     - ダッシュボード（House）、投稿（PaperPlaneTilt）、カレンダー（CalendarBlank）、受信トレイ（Tray）、使用量（ChartBar）、Skills（Package）、チャット（ChatCircle）、設定（GearSix）
     - Phosphor Icons を使用する
     - モバイル（1024px 未満）では drawer（DaisyUI の drawer コンポーネント）に変化する
     - 現在のパスに応じてアクティブ状態を表示する
   - `Header.tsx`: ワークスペース名、通知ベル（Phosphor Bell）、ユーザーアバター + ドロップダウン

4. **ルートレイアウト** `app/layout.tsx`
   - AppShell をラップする
   - DaisyUI テーマを `data-theme="sns-agent"` で適用する
   - メタデータ: title "SNS Agent"

5. **ダッシュボードレイアウト** `app/(dashboard)/layout.tsx`
   - AppShell を使用するレイアウトグループ
   - 認証チェックのプレースホルダ（実際の認証は後続タスクで実装）

6. **レスポンシブ確認**
   - 1280px 以上（デスクトップ）: Sidebar が常時表示
   - 1024px 未満（タブレット/モバイル）: Sidebar が drawer に変化、ハンバーガーメニューで開閉
   - 375px 以上で操作可能であること
