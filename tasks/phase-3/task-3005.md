# Task 3005: Web UI - ダッシュボード画面

## 目的

ダッシュボード画面を実装する。投稿数・予約数・使用量のサマリカード、最近のアクティビティリスト、SNS 別のステータス概要を表示する。

## scope

- `apps/web/src/app/(dashboard)/page.tsx` - ダッシュボードページ
- `apps/web/src/components/dashboard/` - ダッシュボード用コンポーネント
- `apps/web/src/lib/api.ts` - サーバーサイド API クライアント初期化

## 実装指示

1. **API クライアント初期化** `lib/api.ts`
   - サーバーコンポーネントから呼べる API クライアントを設定する
   - SDK の SnsAgentClient をラップし、サーバーサイドの API URL と認証情報を注入する
   - Next.js の server action / RSC から利用可能にする

2. **ダッシュボードページ** `app/(dashboard)/page.tsx`
   - Server Component として実装する
   - API からデータを取得し、各コンポーネントに渡す

3. **サマリカード** `components/dashboard/SummaryCards.tsx`
   - 4 つのカード: 総投稿数、予約待ち数、今月の使用量（推定コスト）、接続アカウント数
   - DaisyUI の stat コンポーネントをベースにする
   - 各カードに Phosphor Icons のアイコンを付ける

4. **SNS ステータス概要** `components/dashboard/PlatformOverview.tsx`
   - SNS ごとのカード（X, LINE, Instagram）
   - 各カードに: 接続アカウント数、最新投稿日時、投稿成功率
   - 未接続の SNS は「未接続」と表示し、接続を促すリンクを表示

5. **最近のアクティビティ** `components/dashboard/RecentActivity.tsx`
   - 直近 10 件のアクティビティ（投稿作成、公開、予約実行、エラー等）をリスト表示
   - 各行: タイムスタンプ、アクション、対象SNS、ステータスバッジ
   - DaisyUI の timeline またはリストコンポーネントを使う

6. **データが空の場合**
   - 初期状態（アカウント未接続、投稿なし）でもエラーにならず、適切な空状態メッセージを表示する
