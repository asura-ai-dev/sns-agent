## 実装指示

1. **usage コマンド** `commands/usage.ts`
   - `sns usage report` - 使用量レポート
     - `--platform <p>` （省略時は全 SNS）
     - `--range <daily|weekly|monthly>` （デフォルト: monthly）
     - `--json`
   - `sns usage summary` - サマリ表示。`--json`

2. **llm コマンド** `commands/llm.ts`
   - `sns llm route list` - ルーティング設定一覧。`--json`
   - `sns llm route set` - ルーティング設定
     - `--platform <p>`
     - `--action <action>`（省略可）
     - `--provider <provider>` （必須）
     - `--model <model>` （必須）
     - `--temperature <t>` （省略可）
     - `--max-tokens <n>` （省略可）
     - `--fallback-provider <p>` `--fallback-model <m>` （省略可）
   - `sns llm route delete <id>` - ルーティング削除

3. **skills コマンド** `commands/skills.ts`
   - `sns skills list` - パッケージ一覧。`--json`
   - `sns skills pack` - パッケージ生成
     - `--platform <p>` （必須）
     - `--provider <llm-provider>` （必須）
   - `sns skills enable <id>` - パッケージ有効化
   - `sns skills disable <id>` - パッケージ無効化
   - `sns skills show <id>` - manifest 表示。`--json`

4. 注: usage / llm / skills の API エンドポイントは Phase 4/5 で実装される。SDK 呼び出しの骨格を実装し、API 未実装時は適切なエラーメッセージを表示する。
