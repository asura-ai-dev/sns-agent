## 実装指示

1. **LINE プロバイダ** `packages/provider-line/src/` を実装する
   - `index.ts`: SocialProvider を実装するクラスを export
   - `auth.ts`: Channel Access Token v2.1 の取得・管理
     - LINE Developers Console での設定を前提とする
     - JWT を使ったトークン発行
   - `post.ts`: メッセージ送信
     - `validatePost`: テキスト 5,000 文字チェック、リッチメッセージ対応確認
     - `publishPost`: Messaging API の push message / broadcast を使って送信
     - `deletePost`: LINE は送信済みメッセージの削除に非対応 -> ProviderError を返す
   - `inbox.ts`: メッセージ受信
     - `listThreads`: ユーザーとのチャットスレッド一覧（Webhook 経由で蓄積したデータ）
     - `getMessages`: スレッド内メッセージ一覧
   - `webhook.ts`: LINE Webhook の署名検証とイベント解析
     - メッセージイベント、フォローイベント等をパース
   - `capabilities.ts`: LINE の capability 定義

2. **capability 定義**

   ```
   textPost: true
   imagePost: true
   videoPost: true
   threadPost: false
   directMessage: true
   commentReply: false
   broadcast: true
   nativeSchedule: false
   usageApi: false
   ```

3. **LINE 固有の考慮事項**
   - LINE はフィード型の投稿がなく、push message / broadcast が「投稿」に相当する
   - broadcast は全友だちへの配信 -> 承認フロー（Phase 6）と連携する設計にしておく
   - リッチメッセージ（Flex Message）は v1 では対応不要だが、拡張ポイントを設けておく

4. `apps/api/src/providers.ts` に LINE プロバイダを ProviderRegistry に追加する

5. `apps/api/src/routes/webhooks.ts` の LINE Webhook エンドポイントで provider-line の `handleWebhook` を呼ぶ
