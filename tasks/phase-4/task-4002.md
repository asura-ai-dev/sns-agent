# Task 4002: provider-instagram - Instagram プロバイダ実装

## 目的

Instagram Graph API / Basic Display API を使用する Instagram プロバイダを SocialProvider インターフェースに準拠して実装する。

- `apps/api/src/providers.ts` - Instagram プロバイダの登録追加

## 実装指示

1. **Instagram プロバイダ** `packages/provider-instagram/src/` を実装する
   - `index.ts`: SocialProvider を実装するクラスを export
   - `auth.ts`: Instagram Graph API の OAuth（Facebook Login 経由）
     - 認可 URL 生成、コード交換、トークンリフレッシュ
     - Instagram はlong-lived token（60日）+ refresh で延長
   - `post.ts`: 投稿操作
     - `validatePost`: テキスト 2,200 文字チェック、画像必須（フィード投稿の場合）チェック
     - `publishPost`: Container 作成 -> メディア公開の 2 ステップ API
     - `deletePost`: Instagram Graph API の DELETE で削除
   - `inbox.ts`: DM / コメント参照
     - `listThreads`: Instagram Messaging API でスレッド一覧
     - `getMessages`: スレッド内メッセージ一覧
   - `webhook.ts`: Instagram Webhook の署名検証とイベント解析
   - `capabilities.ts`: Instagram の capability 定義

2. **capability 定義**

   ```
   textPost: false      // Instagram はテキストのみ投稿不可
   imagePost: true
   videoPost: true      // リール
   threadPost: false
   directMessage: true
   commentReply: true
   broadcast: false
   nativeSchedule: false
   usageApi: false
   ```

3. **Instagram 固有の考慮事項**
   - フィード投稿は画像必須。テキストのみの投稿は ValidationError で拒否する
   - Container 作成 -> 公開の非同期フローを内部で処理する
   - ストーリーズ、リールは v1 では部分的対応（リールのみ可能な場合は対応）
   - レート制限: API Usage の rate limit ヘッダを解析する

4. `apps/api/src/providers.ts` に Instagram プロバイダを ProviderRegistry に追加する

5. `apps/api/src/routes/webhooks.ts` の Instagram Webhook エンドポイントで provider-instagram の `handleWebhook` を呼ぶ
