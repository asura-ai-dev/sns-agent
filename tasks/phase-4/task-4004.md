## 実装指示

1. **予算ポリシー評価ロジック** `core/policies/budget.ts`
   - `evaluateBudgetPolicy(workspaceId, platform, endpoint, additionalCost)`: 予算ポリシーを評価する
     - 該当するポリシーを scope の優先度順（endpoint > platform > workspace）で検索
     - 現在の期間の消費額を usage_records から集計
     - 追加コストを加算して上限と比較
     - 結果: `{ allowed: boolean, action: 'warn'|'require-approval'|'block', consumed, limit, percentage }`
   - 超過前の警告: 80% 到達で warn を返す（ポリシーの action に関わらず）

2. **予算ユースケース** `core/usecases/budget.ts`
   - `listPolicies(workspaceId)`: ポリシー一覧
   - `createPolicy(input)`: ポリシー作成
   - `updatePolicy(policyId, input)`: ポリシー更新
   - `deletePolicy(policyId)`: ポリシー削除
   - `getBudgetStatus(workspaceId)`: 全ポリシーの現在消費状況を返す

3. **予算 API エンドポイント** `routes/budget.ts`
   - `GET /api/budget/policies` - ポリシー一覧（RBAC: budget:read -> admin 以上）
   - `POST /api/budget/policies` - ポリシー作成（RBAC: budget:manage）
   - `PATCH /api/budget/policies/:id` - ポリシー更新（RBAC: budget:manage）
   - `DELETE /api/budget/policies/:id` - ポリシー削除（RBAC: budget:manage）
   - `GET /api/budget/status` - 現在の消費状況（RBAC: budget:read）

4. **投稿公開時の予算チェック統合**
   - `publishPost` ユースケース内で `evaluateBudgetPolicy` を呼ぶ
   - `block` の場合は BudgetExceededError を投げて公開を中止する
   - `require-approval` の場合は承認リクエストを作成する（承認フロー自体は Phase 6）
   - `warn` の場合は公開を続行し、警告情報をレスポンスに含める
