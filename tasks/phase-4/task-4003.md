# Task 4003: 使用量記録と集計 API

## 目的

SNS API / LLM API の使用量を記録するユースケースと、日次・週次・月次で集計する API を実装する。

## 実装指示

1. **使用量ユースケース** `core/usecases/usage.ts` を実装する
   - `recordUsage(input)`: 使用量レコードを記録する
     - input: workspaceId, platform, endpoint, actorId, actorType, success, estimatedCostUsd
   - `getUsageReport(workspaceId, filters)`: 使用量レポートを集計して返す
     - filters: platform, period(daily/weekly/monthly), from, to
     - 集計項目: requestCount, successCount, failureCount, totalEstimatedCost
     - platform 別に集計する
   - `getUsageSummary(workspaceId)`: ダッシュボード用のサマリを返す
     - 今月の合計コスト、今月のリクエスト数、成功率、SNS 別の内訳

2. **UsageRepository 実装** `repositories/usage-repository.ts`
   - `record(usageRecord)`: レコード挿入
   - `aggregate(workspaceId, filters)`: 集計クエリ
     - GROUP BY platform, 期間（日/週/月）で集計
     - SUM(request_count), SUM(CASE WHEN success THEN 1 ELSE 0 END), SUM(estimated_cost_usd)
   - `getSummary(workspaceId, period)`: 期間内のサマリ

3. **使用量 API エンドポイント** `routes/usage.ts` を実装する
   - `GET /api/usage` - 使用量レポート（RBAC: usage:read）
     - クエリパラメータ: platform, period(daily/weekly/monthly), from, to
     - レスポンス: `{ data: { period, platform, requestCount, successRate, estimatedCost }[] }`
   - `GET /api/usage/summary` - サマリ（RBAC: usage:read）
     - レスポンス: `{ data: { totalCost, totalRequests, successRate, byPlatform: {...} } }`

4. **自動使用量記録ミドルウェア** `middleware/usage-recorder.ts`
   - Provider 呼び出し時に自動で使用量を記録する仕組み
   - Provider のメソッドをラップするか、ユースケース層で記録する（ユースケース層での記録を推奨）
   - 推定コストはプラットフォーム・エンドポイントごとの固定単価テーブルを `packages/config` に持つ

5. **推定コスト単価** `packages/config/src/cost-table.ts`
   - SNS API / LLM API の推定コスト単価を定義する
   - 例: X tweet POST = $0.001, LINE push = $0.0001, OpenAI GPT-4o = $0.01/1K tokens
   - v1 では概算値とし、正確な料金は v1.5 で対応する
