# SNS Agent Language Rules

## 目的

SNS Agent の UI 文言は、editorial トーンを保ちながら「英語の見出し層」と「日本語の本文層」を分離して扱う。
このルールは後続 issue で各ページへ適用するための判断基準であり、まずは共通辞書と運用基準を固定する。

## 基本原則

- 英語を使う箇所:
  section eyebrow、kicker、ナビゲーション項目、カードやパネルのラベル見出し
- 日本語を使う箇所:
  本文説明、ボタンラベル、フォームラベル、placeholder、バリデーションエラー、toast、empty state の説明
- 英語は短く識別的に書く:
  UI の骨格を示すためのラベルとして使い、文章化しすぎない
- 日本語は操作意図がすぐ分かる表現にする:
  ユーザーが迷わず操作できることを優先する

## 英語を使う対象

- 英語の eyebrow:
  `Operations Ledger`, `Dispatch Roster`, `Message Queue` のような短い見出し
- 英語のナビゲーション:
  サイドバーや設定内カテゴリの識別名として使う
- 英語のセクションラベル:
  表やカードの上に置く小さな見出しとして使う

## 日本語を使う対象

- 日本語の本文:
  状況説明、補足、空状態、注意文を自然な日本語で書く
- 日本語の操作ラベル:
  保存、キャンセル、削除、再試行など、結果がすぐ分かる言葉を使う
- 日本語の入力支援:
  フォームラベル、placeholder、エラーメッセージは日本語で統一する

## 用語統一

| 概念 | 推奨表記 | 避けたい揺れ |
| --- | --- | --- |
| Post | 投稿 | ポスト, 配信 |
| Draft | 下書き | ドラフト |
| Schedule | 予約 | スケジュール, 予約投稿 |
| Publish | 公開 | 投稿実行, 配信開始 |
| Inbox | 受信トレイ | インボックス |
| Retry | 再試行 | リトライ |
| Save | 保存 | セーブ |
| Delete | 削除 | 消去 |

## 運用メモ

- 英語と日本語を同じ UI 要素に混在させない
- 英語は構造、日本語は操作と説明、という役割分担を崩さない
- 新しい文言を追加するときは、まず英語の見出し語か、日本語の本文語かを判定してから辞書に追加する
