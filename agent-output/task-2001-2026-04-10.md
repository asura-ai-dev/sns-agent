# Handoff: Task 2001 - 認証・セッション管理と RBAC ミドルウェア

## Spec Alignment

spec.md P0-2「認証・権限管理（RBAC）」および AC-21「ロールに応じて操作可否が制御される」に対応。6ロール RBAC の実行基盤（認証ミドルウェア + 権限チェックミドルウェア）を API サーバーに組み込んだ。

## Phase

Implementation (Phase 2)

## completed

- 認証ミドルウェア `apps/api/src/middleware/auth.ts`: Bearer API キー認証（SHA-256 ハッシュで DB 検索 → AgentIdentity 解決）、セッション認証（X-Session-User-Id ヘッダ → User 解決）、未認証時 401 レスポンス
- RBAC ミドルウェア `apps/api/src/middleware/rbac.ts`: `requirePermission(permission)` ファクトリ関数、core の `checkPermission()` を利用、権限なし時 403 レスポンス
- 認証ユースケース `packages/core/src/usecases/auth.ts`: `resolveActorByApiKey()`, `resolveActorByUserId()`, `Actor` 型定義
- User Repository `packages/db/src/repositories/user-repository.ts`: findById, findByEmail, findByWorkspace, create, update
- AgentIdentity Repository `packages/db/src/repositories/agent-identity-repository.ts`: findById, findByApiKeyHash, findByWorkspace, create, update, delete
- Seed スクリプト `packages/db/src/seed.ts`: デフォルトワークスペース、owner ユーザー、agent identity の作成
- `apps/api/src/app.ts` に認証ミドルウェアを全保護ルートに適用（health, webhooks を除外）
- `AppVariables` に `actor: Actor` 型を追加
- 全パッケージのバレルエクスポート更新（middleware/index.ts, repositories/index.ts, db/index.ts, core/index.ts）
- `pnpm run build` 全 13 タスク成功

## 重要な判断

- API キーのハッシュ化に SHA-256 を採用。DB には平文 API キーは保存せず、認証時に受け取った API キーをハッシュ化して比較する
- セッション認証は v1 の簡易実装として `X-Session-User-Id` ヘッダを使用。Web UI は Next.js の server-side rendering からこのヘッダを付与する想定
- 認証ミドルウェアはパスベースで適用（`/api/health`, `/api/webhooks` を除外）
