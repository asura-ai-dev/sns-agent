## known_gaps

- v1 では skill 経路では budget policy / approval flow は通っていない (`postDeps` に `budgetPolicyRepo` / `approvalRepo` を渡していない)。スキル経由の create_post(publishNow=true) は予算チェックをスキップする。Phase 6+ で必要なら追加検討
- `list_posts` の skill 引数は `status`(単数) / `limit` / `platform` のみ対応。`statuses[]` などの複数指定は未対応 (テンプレートには無いため必要十分)
- `accountName` 解決は完全一致のみ。displayName が重複している場合は最初に見つかったものを返す (Drizzle 実装側の `findByWorkspace` の順序に依存)
- skill template の `schedule_post` には `accountName` + `text` + `scheduledAt` のみがあり、create→schedule のアトミック性は保証されない (createPost 成功後に schedulePost が失敗すると draft が残る)。Phase 6+ でクリーンアップが必要なら検討

## 重要判断

- **postDeps に approvalRepo / budgetPolicyRepo を渡さない**: skill 経路は既に gateway 側で approval-required モードで通るため、二重の require-approval を避ける。実投稿のとき予算チェックを通したい場合は別チケットで明示的に追加すべき
- **schedule_post は 2 段呼び出し**: schedulePost は postId 必須なので、まず createPost(draft) → schedulePost の順。skill template の引数 (`accountName`, `text`, `scheduledAt`) を尊重
- **accountName 解決方針**: id 一致 → displayName 完全一致。LLM が出力する `accountName` は人間向けの名前 (displayName) を期待するが、agent から直接 ID を渡されるケースも吸収する
- **createdBy に actor.id を入れる**: 監査ログとの紐付けのため。SkillActor.id をそのまま使用
