## 未完了・既知のギャップ

- **下書き編集画面 `/posts/[id]/edit`** は本チケットの scope 外。PostList の編集ボタンは `/posts/${post.id}/edit` にリンクしているが、そのページは別チケットで実装する想定
- **メディアアップロード実体** — 現状は `URL.createObjectURL` による blob プレビューのみ。`contentMedia` に blob URL が入ったまま `POST /api/posts` に送信される。これは v1 ではチケット指示通り「プレースホルダ可」なので OK だが、実アップロードは後続フェーズで差し替えが必要
- **検索 debounce** — 現在は `onChange` 毎に即 fetch する最小実装。将来 300ms 程度の debounce を入れるのが望ましい
- **ページネーションの URL 同期** — 現状は in-memory state のみで URL にフィルタ/ページが反映されない。共有用 URL を持ちたくなったら `useSearchParams` で拡張できる
- **本体 `next build` 実行** — 並行 agent の `next dev` が同じ `.next/` を共有していた影響で本番ビルドのトレース段階（`.next/server/app/_not-found/page.js.nft.json`）に race condition の ENOENT が発生。`Compiled successfully` / `Generating static pages (11/11)` までは通っているため、成果物自体は問題ない。並行作業が落ち着いたタイミングで `rm -rf .next && next build` を再実行するとクリーンに通るはず

## 重要判断

- **既存 API のエンドポイント形状に追従**：`/api/posts` は既に Task 2004 で実装済み。`platform`/`status` は複数出現または `,` 区切りに対応しており、本 UI はクエリ複数出現形式を採用（`URLSearchParams.append`）
- **デザイン一貫性優先**：task-3008 の `PlatformIcon` / settings の紙面トーンと合わせ、`Fraunces` display + `DM Sans` body、warm off-white、罫線ベース。SNS 模倣は避けプラットフォーム色は chip と縦罫線のみに制限
- **API 未起動時の fallback**：投稿一覧は「エラーカード + 再読み込みで再試行」と縮退、作成フォームは `AccountSelect` に同じエラーカードを出して送信ボタンを disable。これはユーザーが空データベース時や API 未接続状態でも UI がクラッシュしないために必須
- **done_when ファイルを読んでいない**：generator 制約に従い `task-3006.done_when.md` は一切参照していない。検証方法は scope + refs から推定した
