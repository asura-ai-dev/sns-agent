# Task 1006: apps/api - Hono API サーバー骨格 - Handoff

## Spec Alignment

spec.md P0-1「モノレポ基盤構築」の API サーバー部分に対応。apps/api を Hono + Node.js で構築し、REST エンドポイント骨格・共通ミドルウェア・ヘルスチェックを実装。design.md セクション 1.2（REST 中心）、1.4（Node.js 実行基盤）、4.4（ミドルウェアチェーン）に準拠。

## completed

- `apps/api/src/index.ts` - @hono/node-server の serve() で API_PORT（デフォルト 3001）起動
- `apps/api/src/app.ts` - Hono インスタンス、ミドルウェア、DB context 注入、ルーティング組み立て
- `apps/api/src/middleware/request-id.ts` - X-Request-Id ヘッダ付与（UUID v4）、既存ヘッダ保持
- `apps/api/src/middleware/error-handler.ts` - DomainError -> エラーレスポンス変換、それ以外 -> 500 SYSTEM_ERROR
- `apps/api/src/middleware/cors.ts` - 開発環境 `*`、本番 WEB_URL のみ
- 11 ルートファイル: accounts, posts, schedules, usage, budget, llm, skills, agent, audit, approvals, webhooks（各 501 ダミー）
- GET /api/health -> { status: "ok" }
- package.json に dev（tsx watch）と build スクリプト
- @sns-agent/config, @sns-agent/core, @sns-agent/db を workspace:\* 依存追加
- uuid を依存追加
