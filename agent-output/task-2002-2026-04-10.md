## completed

- アカウント管理ユースケース `packages/core/src/usecases/account.ts`
  - `listAccounts(deps, workspaceId)`: ワークスペースのアカウント一覧（トークンなし、tokenExpiryWarning 付き）
  - `getAccount(deps, accountId)`: アカウント詳細（NotFoundError を投げる）
  - `initiateConnection(deps, workspaceId, platform)`: Provider の connectAccount を呼び、state を暗号化して認可 URL を返す
  - `handleOAuthCallback(deps, code, state)`: state を復号、Provider に code を渡してトークン取得、暗号化して保存
  - `disconnectAccount(deps, accountId)`: status を 'revoked' に変更（論理削除）
  - `refreshAccountToken(deps, accountId)`: Provider の refreshToken を呼び、新トークンで更新。失敗時は status='error' に
  - `checkTokenExpiry(deps, workspaceId)`: 7 日以内に期限切れ or 既に expired/error のアカウントを返す
- 型定義: `AccountSummary`（トークンなし）、`AccountUsecaseDeps`（DI コンテキスト）、`OAuthStatePayload`（state の中身）
- API エンドポイント `apps/api/src/routes/accounts.ts`（骨格 → 実装に置換）
  - `GET /api/accounts` - listAccounts（RBAC: account:read）
  - `POST /api/accounts` - initiateConnection（RBAC: account:connect、body: `{platform}`、resp: `{data:{authUrl}}`）
  - `GET /api/accounts/callback` - handleOAuthCallback（認証不要、state/code を query から取得）
  - `GET /api/accounts/expiring` - checkTokenExpiry（RBAC: account:read）
  - `GET /api/accounts/:id` - getAccount（RBAC: account:read）
  - `DELETE /api/accounts/:id` - disconnectAccount（RBAC: account:connect）
  - `POST /api/accounts/:id/refresh` - refreshAccountToken（RBAC: account:connect）
- トークン暗号化保存: `core/domain/crypto.ts` の encrypt/decrypt を使用。credentialsEncrypted カラムに暗号文を格納。レスポンスには一切含めない
- トークン有効期限管理: `tokenExpiresAt` を保存、`tokenExpiryWarning` フラグ（7日以内）を AccountSummary に付与
- OAuth state セキュリティ: workspaceId/platform/nonce を暗号化して state パラメータに埋め込み、コールバックで復号・検証
- `apps/api/src/app.ts`: callback ルート（`/api/accounts/callback`）のみ認証ミドルウェアをバイパスする条件分岐を追加
- `apps/api/src/types.ts`（新規）: `AppVariables` 型を切り出し（accounts ルートから app.js を import すると循環依存になるため）
- core の barrel export 更新: `listAccounts`, `getAccount`, `initiateConnection`, `handleOAuthCallback`, `disconnectAccount`, `refreshAccountToken`, `checkTokenExpiry`, `AccountSummary`, `AccountUsecaseDeps`, `OAuthStatePayload`
- ユニットテスト `packages/core/src/usecases/__tests__/account.test.ts`（18 ケース、全 pass）
- Vitest を core パッケージに導入（`pnpm add -D vitest`、package.json の test script 更新、tsconfig で test ファイル除外）

## known_gaps

- **Provider 実装は空**: `packages/provider-x/src/index.ts` は `export {};` のみ。実際の OAuth 2.0 PKCE フローや X API 呼び出しは未実装。`accounts.post("/", ...)` や callback は「Provider が未登録」でエラーになる状態。これは別チケット（provider-x の実装）で解決する想定
- **Provider DI 未接続**: `buildDeps()` では `providers: new Map()` を返しており、ルートハンドラは実質的に provider があれば動くが現時点では動作しない。provider-x タスク完了後に `buildDeps()` でインスタンスを map に詰める必要がある
- **リフレッシュ時の復号済みトークン引き渡し**: `provider.refreshToken(accountId)` というシグネチャだが、本来は credentials を復号して渡すべき。現状は accountId のみ渡しており、provider 側で repository から取得する前提。interface を見直すべきかもしれない
- **db パッケージの既存ビルド失敗**: `packages/db/src/repositories/audit-log-repository.ts:42` で Drizzle の `eq` 型エラー（`actor_type` enum の型不整合）が存在。これは Task 2002 の scope 外（Task 1004 / Task 2001 あたりで発生した既存の問題）。本タスクでは触らず、別チケットで対応
- **ENCRYPTION_KEY デフォルト値**: 開発用デフォルト値（`0123456789abcdef...`）を routes/accounts.ts にハードコード。本番環境では環境変数が必須（config/env.ts で production 時の required 判定あり）だが、dev で fallback が効く状態。config/env.ts の getConfig() 経由に切り替える余地あり
- **監査ログ統合なし**: アカウント接続・切断時に AuditLog への記録は未実装（Phase 6 の監査フェーズで対応予定）
