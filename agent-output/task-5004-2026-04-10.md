# Task 5004 Handoff — Web UI チャットインターフェース

## Spec Alignment

- spec.md AC-16: Web UI のチャット画面から LLM を呼び出し、投稿作成等の操作を指示できる
- spec.md AC-17: チャットからの操作実行前にプレビューが表示され、承認後に実行される

design.md 10.2（チャット画面）と architecture.md 9.2（Web UI チャットから操作まで）で定義された
2 ペイン + skill 実行承認フローに対応する Web UI を実装した。Agent Gateway
(Task 5002) の `POST /api/agent/chat` / `POST /api/agent/execute` /
`GET /api/agent/history` を前提にした client integration。API 未起動時は
fallback 経路でデモ会話と proof sheet を描画し、UI 全体が常に動作する状態を維持する。

## 既知のギャップ / 課題

- **SSE は「疑似」ストリーミング**: 現行 Agent Gateway (`apps/api/src/routes/agent.ts` Task 5002 実装) は
  非ストリーミング JSON を返すため、`streamChatMessage` は fetch 完了後に
  14ms/chunk でトークン配信する疑似実装となっている。API 側が本物の SSE（text/event-stream）に
  対応したら、`streamChatMessage` の try ブロックだけを `ReadableStream` + `TextDecoder` に置換すれば良い
  （呼び出し側の callback shape は固定済み）。design.md 1.5 でも "Web UI チャットは SSE でストリーミング"
  と明記されているため、v1.5 で API 側を更新するチケットが必要
- **executeInvoker は placeholder**: apps/api/src/routes/agent.ts の `placeholderInvoker` は
  常に `status: "deferred"` を返す。Task 5003+ で core use case へのルーティング実装予定
  （UI 側は deferred/executed いずれも press log 表示済み）
- **Next.js app に /api proxy route はない**: apps/web は apps/api (Hono) に直接相対パスで fetch する。
  開発環境で apps/api を起動していない場合、`!res.ok` → fallback 経路で動作する設計。
  本番では `NEXT_PUBLIC_API_BASE_URL` を API の絶対 URL にする運用
- **モバイル device emulation 未検証**: agent-browser で iOS シミュレータ起動不可だったため
  目視確認は lg+ viewport のみ。responsive CSS（`lg:hidden` / `lg:block`, `mobilePane` state）は
  コード上で実装済みだがブラウザ上での確認は未実施
- **履歴の完全再生は未実装**: ConversationList から過去の会話を選択しても
  ChatContainer は `seedMessages` 無しで再初期化される。history API は集約済みの summary
  のみで生 messages を返さない（audit_logs 由来）ため、完全な transcript を復元するには
  API 側に conversation-aware な messages エンドポイントを追加する必要がある

## 重要判断

- **Editorial トーン維持 (The Wire Room)**: 既存ダッシュボード (`page.tsx`) の Fraunces + DM Sans /
  wire-offline banner / double rule / mono meta-strip という新聞紙面トーンを全面踏襲。
  チャット UI を「電報デスク」メタファに寄せることで generic な AI chat の見た目を回避
- **Fallback を 1st class**: API が起動していなくても UI 全体が完結する（demo history +
  demo proof sheet + press log）。これにより design review / evaluate が API 依存なしに可能
- **skill 承認フローのスクロール位置**: ActionPreview は AI message の「直後」に差し込む
  （最後尾ではなく anchor message の次）。これにより長い会話でも提案と proof sheet の
  文脈が一目で繋がる
- **Enter キー送信の IME 対応**: `e.nativeEvent.isComposing` を見ることで日本語変換中の
  Enter で誤送信しないようにした。Shift+Enter は常に改行
- **`emitAsTokens` のチャンクサイズ**: 1 文字ずつだと遅すぎ、10+ 文字だと teletype 感が薄い。
  3 文字 / 14ms に落ち着かせた（~70 chunks/sec で読みやすさと速度を両立）
- **PostToolUse hook でフォーマット済み**: 3 ファイル（ActionPreview/ConversationList/ChatContainer）が
  Write 後に hook で整形された。再編集時は先に Read する必要あり
