# Task 3001: packages/cli - CLI 基盤と accounts コマンド

## Spec Alignment

本チケットは spec.md の **AC-7**（`sns accounts list --json` で JSON 出力）および
**AC-10**（全コマンドが非対話で完結し、終了コード 0/1 を返す）に対応する。
さらに AC-1（接続状態を CLI から確認できる）の CLI 側経路の初回実装でもある。

## 概要

`packages/cli` に commander ベースの CLI 基盤を構築し、`sns accounts` サブコマンド
4 つ（list / show / connect / disconnect）を実装した。SDK の `SnsAgentClient` を
経由して API を呼び出し、`--json` フラグでフォーマッター（JSON / table）を切り替える。

## 実装ファイル

- `packages/cli/package.json` - `bin: { "sns": "./dist/index.js" }`、commander / @sns-agent/sdk 依存
- `packages/cli/tsconfig.json` - `types: ["node"]` と `lib: ["ES2022","DOM"]` 追加
- `packages/cli/src/index.ts` - エントリポイント（shebang、commander セットアップ、終了コード制御）
- `packages/cli/src/config.ts` - 設定読み込み（CLI > env > ~/.sns-agent/config.json > default）
- `packages/cli/src/context.ts` - `buildContext` / `runCommand`（共通エラー処理、exit code 1）
- `packages/cli/src/errors.ts` - SdkError を人間可読に変換（401/403/404/422/429/5xx）

- `packages/cli/src/formatters/json.ts` - JSON 出力（成功・エラー・info 全て stdout）
- `packages/cli/src/formatters/table.ts` - テーブル / key-value / stderr エラー
- `packages/cli/src/formatters/index.ts` - `selectFormatter(json)` ファクトリ
- `packages/cli/src/commands/accounts.ts` - accounts サブコマンド 4 種

## Spec/Design との対応

- **AC-7**: `sns accounts list --json` を実装。モック API に対し `{ data: [...] }` を stdout に整形出力することをローカル検証済み
- **AC-10**: 全コマンドが非対話。成功時 exitCode 0、例外時 `runCommand` で exitCode 1 を設定。設定不備（API key 欠如）・invalid platform・API 404・fetch 失敗のいずれも exit 1 で検証済み
- **design.md §2**: `packages/cli/src/{commands,formatters,index.ts}` のディレクトリ構成を踏襲
- **design.md §5**: `cli -> sdk, config` の依存関係を遵守（@sns-agent/sdk のみ workspace 依存追加）
- **requirements.md §7.5**: グローバル `--json` / `--api-url` / `--api-key`、環境変数 `SNS_API_URL` / `SNS_API_KEY`、`~/.sns-agent/config.json` の優先度を実装

## 設定の優先度（実装）

1. CLI 引数（`--api-url` / `--api-key`）
2. 環境変数（`SNS_API_URL` / `SNS_API_KEY`）
3. `~/.sns-agent/config.json`（JSON。`apiUrl` / `apiKey` を読む）
4. デフォルト（`apiUrl=http://localhost:3001`、`apiKey=""`）

`apiKey` が解決できない場合は `CONFIG_MISSING_API_KEY` エラーで exit 1 を返し、
`--api-key` / `SNS_API_KEY` / config.json のいずれかを設定するよう案内する。

## エラーハンドリング

- SDK の `SdkError` を `normalizeError()` で `{ code, message, details }` に変換
- ステータスコード別の人間可読メッセージ（401: auth、403: forbidden、422: validation、429: rate limit、5xx: server error 等）
- `--json` モード: エラーを `stdout` に JSON `{ error: { code, message, details } }` で書く
- テーブルモード: エラーを `stderr` に `Error [CODE]: message` で書く
- 全エラー経路で `process.exitCode = 1` → `process.exit` を保証
