## 重要判断

- **`--account` の解決**: UUID 風 (`/^[0-9a-fA-F-]{16,}$/`) を id 扱い、そうでなければ `accounts.list` の `platform + displayName` で検索。spec.md AC-4 の「`--account main`」のような name 指定を機能させるために必要。複数一致は曖昧エラーを返す
- **`--text` と `--file` の排他**: 両指定を許すと意図不明瞭になるため VALID_CONFLICT エラーで拒否
- **inbox は SDK 汎用メソッド使用**: SDK に inbox リソースが無い。チケット指示どおり汎用 get で実装し、skills.ts と同じパターンに揃えた
- **`schedule show` も汎用メソッド**: 既存 SDK の `SchedulesResource` に get メソッドが無いため、チケット指示（汎用 get 許容）に従い `client.get` で実装
- **`--media` の MIME 推定**: CLI 側で拡張子から推定。誤判定時の保険として image/png / video/mp4 をデフォルトにした
- **ISO 日時バリデーション**: `new Date(value)` のパース結果が NaN でないことのみ確認する緩いバリデーション。タイムゾーン形式の厳密チェックは過剰とみなした
