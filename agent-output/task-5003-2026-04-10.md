## 重要な設計判断

- **core ⇄ skills の循環依存回避**: core/usecases/skills.ts は `@sns-agent/skills` を import しない。代わりに `SkillManifestBuilder` を DI 受けし、apps/api 側で `@sns-agent/skills` の `generateSkillPackage` をラップして渡す。これにより `packages/core` -> `packages/skills` -> `packages/core` の循環を断ち切る (agent-gateway.ts と同じパターン)。
- **冪等な generate**: `POST /api/skills/generate` を再実行しても `(workspaceId, name)` で既存を update し、enabled 状態を保持する。CLI `sns skills pack ...` の再実行で重複行が出ない。
- **capability ベースのアクション選択**: ProviderRegistry から capability を取り、`textPost=false` の provider では `create_post` / `schedule_post` を自動的に除外する。LINE のように broadcast 特化の provider が登録されたケースでも安全に degrade する。
- **workspaceId スコープの徹底**: enable / disable / getSkillManifest はすべて actor.workspaceId と既存 row の workspaceId が一致しない場合 NotFoundError。情報漏洩を防ぐ。
